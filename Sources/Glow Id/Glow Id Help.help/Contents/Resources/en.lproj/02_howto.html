How to Use

Opening a File

Method 1: Double-Click (Recommended)

Once Glow Id has been launched at least once, .indd / .indt / .indb / .indl / .idml files are automatically associated with it. From then on, simply double-clicking a file launches Glow Id, detects the version, and opens the file with the appropriate InDesign.

  1. Double-click a .indd / .indt / .indb / .indl / .idml file.
  2. Glow Id launches and detects the file's version.
  3. If a matching InDesign is found, the file opens automatically. Otherwise, the Notification Window appears.
Tip If double-clicking does not launch Glow Id, check whether Associate file types on launch is turned off in Preferences. Turning it on immediately re-associates the file types with Glow Id.

Method 2: Drop onto the Dock Icon

  1. Drag and drop a file onto the Glow Id icon in the Dock.
  2. Glow Id detects the version and launches the matching InDesign automatically.
  3. If no matching version is found, or if manual selection is required, the Notification Window appears.

Method 3: Open from the Menu

  1. Choose File > Open… (⌘O) from the menu bar.
  2. Select one or more files in the file chooser dialog.

IDML Files

IDML is a format intended for opening files in a different, older version of InDesign, so the Notification Window always appears. No warning is shown. Select the version of InDesign you want to open the file with.

Processing Multiple Files at Once

You can open multiple files at the same time. Glow Id processes them one by one. If the Notification Window appears, each window is shown in sequence — the next window appears after you close the current one.

Holding ⌘ While Opening

Normally, when a matching InDesign version is found, the file opens automatically. Holding ⌘ (Command) while opening a file forces the Notification Window to appear, even when a matching version exists.

Glow Id Quits Automatically

After handing the file off to InDesign, Glow Id quits on its own. There is no need to quit Glow Id manually while working in InDesign.

Tip Use the "Quit" button in the Notification Window to quit the currently running InDesign directly from Glow Id.

First Launch After Install or Update

After installing Glow Id or replacing it with an updated version, always launch Glow Id at least once.

Replacing the app with an update can break the file association for .indd / .indt / .indb / .indl / .idml files. Simply launching Glow Id restores the association automatically.

In addition, Glow Id automatically imports file icons from InDesign on launch when either of the following applies:

  • The highest installed version of InDesign (excluding Beta and Prerelease) has changed.
  • Glow Id itself has been updated to a new version.

Icon import happens only when Glow Id is launched from a writable location such as the Applications folder. If you launch it directly from where it was unzipped (for example, the Downloads folder), the import cannot proceed and a message such as “Please move Glow Id to the Applications folder” is shown. In that case, move Glow Id to the Applications folder (or, on a non-administrator account, the Applications folder in your home folder) and launch it again.
