Preferences

Open the Preferences window from the menu bar: Glow Id > Preferences…

Always show Notification Window

When enabled, the Notification Window always appears — even when a perfectly matching version of InDesign is found — so you can choose the app manually every time.

Default: Off (opens automatically when a matching version is found)

Tip To show the Notification Window just once without changing this setting, hold ⌘ (Command) while opening the file (or choose Open from the right-click menu).

Associate file types on launch (.indd / .indt / .indb / .indl / .idml)

Default: On

When on: Every time Glow Id launches, it re-registers itself as the default app for .indd / .indt / .indb / .indl / .idml files. Double-clicking those files will launch Glow Id. If another app changes the association, Glow Id automatically reclaims it on the next launch.

When turned off: At the moment you turn it off, the association is changed to the highest installed version of InDesign (including Beta and Prerelease). Double-clicking those files will launch that InDesign directly, bypassing Glow Id. Use this setting when you want to restore the normal InDesign file association without going through Glow Id.

Tip If double-clicking no longer launches Glow Id, check whether this option has been turned off. Turning it back on immediately re-associates the file types with Glow Id.

InDesign Document (.indd) Detection Version

Controls how version information is read from .indd / .indt files and how it is displayed in the Notification Window.

Major + Minor (Header)
Reads and displays the version as two digits — for example, "20.5". Only the beginning of the file is read, so processing is fast. This is sufficient for everyday use.
For CS6 (8) through CC 2018 (13), the full version is always retrieved internally, since the minor version may be affected by a bug that prevents it from updating correctly. The Notification Window still displays only the major.minor format.
Full Version (Document History)
Reads and displays all four version digits — for example, "20.5.0.123". This requires a deeper analysis of the file's internal structure and takes slightly longer. Use this when you need to verify the exact build number.

This setting only affects .indd and .indt files. The display for .indb, .indl, and .idml files is not affected.

Version Matching

How the file's creation version is compared against installed InDesign apps depends on the file type.

Document (.indd) / Template (.indt)
Matched by major.minor version. Regardless of the Detection Version setting above, matching always uses major.minor.
Book (.indb)
Matched by major.minor version.
Library (.indl)
Matched by major.minor version by default. For CS6 (8) through CC 2018 (13), matched by major version only.
IDML (.idml)
No version matching. The Notification Window always appears so you can choose which version to open.
