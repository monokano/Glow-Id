環境設定

設定ウインドウは、メニューバー「Glow Id」→「設定…」から開きます。

常に通知ウインドウを表示する

オンにすると、バージョンが完全に一致する InDesign が見つかった場合でも、自動的に開かず必ず通知ウインドウを表示します。

デフォルト：オフ（一致するバージョンがあれば自動で開く）

ヒント 設定を変えずに一時的に通知ウインドウを表示したい場合は、⌘（Command）キーを押しながらファイルをドロップ（または右クリックメニュー「開く」を選択）してください。

起動時に関連付けをする（.indd / .indt / .indb / .indl / .idml）

デフォルト：オン

オンの場合：Glow Id を起動するたびに、.indd / .indt / .indb / .indl / .idml ファイルを Glow Id に関連付けます。ファイルをダブルクリックすると Glow Id が起動するようになります。他のアプリが関連付けを変更しても、次回 Glow Id を起動した時点で自動的に復元されます。

オフにした場合：オフに切り替えた時点で、関連付けがインストール済みの最上位バージョンの InDesign（Beta・Prerelease 含む）に変更されます。ファイルをダブルクリックすると、その InDesign が直接起動します（Glow Id は経由しません）。Glow Id を使わずに通常の InDesign の関連付けに戻したい場合にオフにしてください。

ヒント ダブルクリックで Glow Id が起動しなくなった場合は、このチェックがオフになっていないか確認してください。チェックをオンするだけで関連付けされます。

InDesign ドキュメント（.indd）の検出バージョン

.indd / .indt ファイルのバージョン取得方法と通知ウインドウでの表示を切り替えます。

メジャー＋マイナー（ヘッダー）
バージョンを「20.5」のように メジャー.マイナー の2桁で取得・表示します。ファイルの先頭部分のみを読み取るため、高速に処理できます。通常の作業ではこの設定で十分です。
CS6 (8) ～ CC 2018 (13) は、マイナーバージョンが更新されないバグの影響が懸念されるので、常にフルバージョンを取得します。通知ウィンドウにはメジャー.マイナー の2桁を表示します。
フルバージョン（ドキュメントヒストリー）
バージョンを「20.5.0.123」のように4桁すべてで取得・表示します。ファイルの内部構造を詳細に解析するため、メジャー＋マイナーより時間がかかります。細かい番号まで確認したい場合に使います。

この設定は .indd / .indt にのみ影響します。.indb / .indl / .idml の検出方法は変わりません。

バージョン一致判定

ファイルの作成バージョンと InDesign アプリのバージョンを比較する方法は、ファイルの種類によって異なります。

ドキュメント (.indd) / テンプレート (.indt)
メジャー.マイナーで一致判定します。検出バージョンの設定に関わらず、比較はメジャー.マイナーで行います。
ブック (.indb)
メジャー.マイナーで一致判定します。
ライブラリ (.indl)
原則メジャー.マイナーで一致判定します。CS6 (8) ～CC 2018 (13) はメジャーのみで一致判定します。
IDML (.idml)
一致判定はしません。常に通知ウインドウが表示され、開くバージョンを選択します。
